# Artifact Schemas

## tool-selection.json

Generated by `scripts/select_tools.py`. It records the user's per-run choice, active providers, unavailable selections, readiness reasons, and provider roles without storing credentials. Treat only entries with `active: true` as callable in this run.

## writing-selection.json

Generated by `scripts/select_writer.py`. It records the explicitly selected report-writing backend and model, supported roles, credential readiness, and the required service-region notice. `agent` means the current host Agent writes directly; any other choice means OpenRouter.

## research-map.md

```markdown
| ID | Subquestion | Type | Status | Claim IDs | Gap |
|---|---|---|---|---|---|
```

Use `required | optional | counterpoint` and `open | partial | covered | blocked`.

## query-log.md

```markdown
| Round | Pack | Query ID | Query | Duty | Provider | Result count | Source IDs | Insight Context | Context influence | Time | Notes |
|---|---|---|---|---|---|---:|---|---|---|---|---|
```

Round 1 uses `initial-brief`. Round 2 onward must use the Context ID produced by `prepare-next` and one of `used | considered-not-used | countered | background-only`. Every value is valid: the field records influence without requiring obedience.

## insights.md

```markdown
## Round N
- Input query IDs:
- Input source IDs:
- Insight I-RN-01:
- Insight I-RN-02:
- New concepts:
- Conflicts/uncertainty:
- Candidate next gap:
- coverage:
- authority_gap:
- open_conflicts:
- perspective_gap:
- systematic_gap:
- high_value_delta:
- New high-value Claim IDs:
```

Every round requires 2–4 Insights. IDs use `I-R<round>-<two digits>`. Insights are mandatory research memory, not factual citations, and neither later searches nor the report must adopt them.

## insight-context.jsonl

Generated by `scripts/insight_gate.py prepare-next`. Each line records delivery of one complete Insight snapshot to the next search round:

```json
{"context_id":"CTX-R2-ab12cd34","from_round":1,"next_round":2,"insight_ids":["I-R1-01","I-R1-02"],"snapshot_sha256":"","packet_file":"round-context/round-02-insight-context.md","delivered_to":"host-search-agent","delivery_mode":"stdout+file","delivered_at":""}
```

The receipt proves context delivery, not that Insights determined the search direction.

## round-context/

Contains one immutable packet per added round with the Context ID, previous-round Insight IDs, snapshot hash, non-directive notice, and complete Insight text. Agentic research tools receive the packet as background; atomic search tools are directed by the host Agent after it reads the packet.

## evidence-ledger.md

```markdown
| Claim ID | Claim | Type | Supporting sources | Opposing sources | Confidence | Verification | Argument unit |
|---|---|---|---|---|---|---|---|
```

Claim type: `fact | source_view | analogy | inference | uncertainty`.
